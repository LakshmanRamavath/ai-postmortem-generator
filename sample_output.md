## Root Cause

Redis primary node ran out of memory due to unbounded cache growth on the
payment session keys. No TTL was set on session entries created during a
high-volume batch processing window earlier that day. When memory was exhausted,
the node rejected new writes and became unresponsive to health checks.

Automatic failover was not configured — the replica was available but required
manual promotion.

---

## Impact

- Payment service degraded for 22 minutes (02:14–02:36 UTC)
- API error rate peaked at 34% during the outage window
- Estimated 4,200 payment requests failed or timed out
- No data loss — all failed transactions were safely retried by clients

---

## Resolution

1. On-call engineer identified Redis memory exhaustion via ClickHouse logs
2. Redis replica manually promoted to primary via `redis-cli REPLICAOF NO ONE`
3. Old primary restarted with memory limits enforced
4. Stale session keys flushed, TTL policy applied to session key prefix
